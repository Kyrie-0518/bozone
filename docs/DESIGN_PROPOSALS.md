## 方案 C：双色深浅切换（推荐）

> **设计哲学**: Atmospheric Shift — 白天暖石，夜晚深墨，一键切换

```
                     Light Mode                    Dark Mode
                ┌─────────────────┐       ┌─────────────────┐
                │  bg: #F7F5F2    │       │  bg: #1C1917    │
                │  card: #FFFFFF  │  ⇄   │  card: #2D2925  │
                │  text: #1C1917  │       │  text: #F7F5F2  │
                │  border: #E5E0D8│       │  border:#3D3832 │
                │  accent: #D97706│       │  accent:#F59E0B │
                └─────────────────┘       └─────────────────┘
```

### 色彩 Token 双模式

| Token | Light | Dark |
|-------|-------|------|
| `--bg-page` | `#F7F5F2` | `#1C1917` |
| `--bg-surface` | `#FFFFFF` | `#2D2925` |
| `--bg-surface-alt` | `#F0EDE8` | `#2D2925` |
| `--text-primary` | `#1C1917` | `#F7F5F2` |
| `--text-secondary` | `#78716C` | `#A8A29E` |
| `--border` | `#E5E0D8` | `#3D3832` |
| `--accent` | `#D97706` | `#F59E0B` |

### 优势（推荐理由）
- **全市场唯一** — 13款竞品中0%支持暗色模式
- 差异化卖点极强
- Light模式继承方案A成熟度
- Dark模式适合长时间运营操作（核心用户场景）
- 技术方案成熟（CSS变量即可实现）

## 推荐结论

**首选方案 C（双色深浅切换）**，原因：

1. 全市场0%支持暗色模式 → 独占性卖点
2. Light 模式保留经典SaaS体验 → 降低学习成本  
3. Dark 模式适合跨境卖家深夜运营 → 真实需求场景
4. From amber warmth to amber glow → 品牌一致性跨越两个模式
5. 技术成本可控，React方案成熟

**初期优先开发 Light 模式，Dark 模式作为 Phase 2 差异化特性发布。**
