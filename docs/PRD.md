# Bozone 功能需求规格文档 v1.0

> **定位**: TikTok Shop 专精 SaaS ERP  
> **设计系统**: Warm Industrial · Plan C 双色深浅切换  
> **技术栈**: React 19 + Vite + shadcn/ui + TanStack Router + Recharts  
> **更新日期**: 2026-05-28

---

## 一、系统总览

Bozone 是一款面向 TikTok Shop 跨境卖家的全链路 SaaS ERP，覆盖店铺、订单、商品、财务、达人、库存、素材、广告 8 大模块。

### 用户角色

| 角色 | 权限范围 |
|------|----------|
| 超级管理员 | 所有功能 + 系统配置 + 用户管理 |
| 运营主管 | 看数据 + 管理订单/商品/达人/素材 |
| 财务 | 财务模块 + 只读其他 |
| 运营专员 | 订单处理 + 达人登记 + 素材上传 |

### 模块 5 — 财务核算 Finance ⭐核心

| 功能点 | 描述 | 优先级 |
|--------|------|:---:|
| **费用项配置** | 用户自定义费用模板（采购/佣金/广告/物流/包装/仓储/退货/其他） | P0 |
| **订单级利润** | 收入 - Σ各项费用 = 净利润，每笔订单自动计算 | P0 |
| **单品级利润** | 按SKU汇总所有订单的利润，费用按配置规则分摊 | P0 |
| 费用分摊规则 | 广告费按订单金额比例分摊 / 达人佣金按实际订单归属 / 物流费按商品重量分摊 | P0 |
| 多货币汇率 | MYR→RMB 转换，支持自定义汇率 + 自动获取 | P0 |
| 利润趋势 | 按天/周/月展示利润变化 + ROI | P0 |
| 财务报表 | 营收表/利润表/费用明细表，可导出 | P1 |
| 对账功能 | 与 TikTok Shop 后台结算报告比对 | P1 |
| 自动计算引擎 | 费用项根据规则自动归集计算（未来升级） | P2 |
| 未结算资金 | 待收款/待确认金额自动统计 + 账期预警 | P2 |

#### 费用项模板设计（用户自定义）

```typescript
interface CostItem {
  id: string
  name: string          // 费用名称，如"达人佣金"
  type: 'fixed' | 'percentage' | 'per_order' | 'per_kg'
  value: number          // 固定金额 / 百分比 / 每单 / 每公斤
  applyTo: 'all' | 'shop' | 'product'  // 适用范围
  autoCalculated: boolean  // 是否自动计算（未来版本）
}
```

## 五、技术要点

| 维度 | 决策 |
|------|------|
| TikTok API | 使用 TikTok Shop Open API 进行订单同步和店铺授权 |
| 利润计算引擎 | 服务端计算，费用模板 → 订单费用明细 → 订单级/单品级利润 |
| 多货币 | JSON 存储汇率配置，计算时实时转换 |
| 数据库 | SQLite（当前），预留 PostgreSQL 扩展 |
| 前端组件 | shadcn/ui + TanStack Table（排序/筛选/分页） |
| 图表 | Recharts（已集成） |
| 状态管理 | Zustand（已集成）+ TanStack Query（已集成） |
| 认证 | JWT + RBAC（取代 Clerk，自建） |
| SaaS | 多租户数据隔离，按店铺计费预留 |
